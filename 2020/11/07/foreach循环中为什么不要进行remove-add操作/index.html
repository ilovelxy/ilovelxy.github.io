foreach循环中为什么不要进行remove/add操作

发表于 2020-11-07 阅读次数： Changyan： 本文字数： 3.4k 阅读时长 ≈ 3 分钟

昨天在看JavaGuide的时候，发现了一个关于foreach循环的一个很有意思的问题。

众所周知，不建议在forEach循环中调用add()/remove()方法，阿里巴巴编程规范里面指明了这一点。那么是为什么呢？相信很多人心中会有这个疑问。

从下面三段代码来分析这个问题：

  1. public static void method1() {
        List<String> list = new ArrayList<>();
        list.add("1");
        list.add("2");
        for (String s : list) {
            System.out.println(s);
            if ("1".equals(s)) {
                System.out.println("111111");
                list.remove(s);
            }
        }
        list.forEach(System.out::println);
    }

编译器确实将foreach循环翻译为了iterator迭代器。首先调用`iterator().hasNext()`方法判断有没有下一个元素。

**注意：**这里的`list.iterator()`方法调用的是`ArrayList`内的方法，返回的是`ArrayList`的一个内部类。

对这个内部类的理解：

内部定义了三个变量：`cursor,lastRet和expectedModCount`。

- `cursor:`游标。当前迭代器所在的位置。在构造函数中没有进行显式初始化，因此初始化为0.
- `lastRet:`上一个返回的元素的索引。默认为-1.
- `exceptedModCount:`期待结构改变的次数。这个变量只有通过内部类的方法才能进行修改。初始时和`modcount`的大小相等。
